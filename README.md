## ✨ 它解决了什么问题？

同一个项目，不同的写法，面试效果天差地别：

```diff
- ❌ "实现了图书的增删改查功能"
+ ✅ "设计并实现高并发图书管理平台，引入Redis缓存层降低数据库负载60%，
+    通过索引优化将查询效率提升300%，支持500+用户同时访问"
```

**你的代码已经做到了**——只是不知道怎么写到简历上。这个 Skill 就是干这个的。

| 缓存一致性如何保证？ | Cache Aside、延迟双删 | 延迟双删的时序问题？ | ⭐⭐⭐⭐ |

## ⚙️ 工作原理

```
你的代码 → 分析技术栈/架构 → 映射面试能力 → 生成量化描述 → 输出简历+面试题
```

三步完成：

1. **读代码**：扫描项目结构，识别技术栈（框架、数据库、缓存、MQ 等）
2. **转能力**：将每个技术点映射为面试能力（如 Redis → 缓存设计能力）
3. **写简历**：按「业务场景 + 技术方案 + 具体动作 + 量化数据」结构生成

## 🧠 设计理念

- **绝不造假**：从真实代码出发，能做的才写，不能做的标记为「可讨论」
- **数据量化**：每条经历都带数据指标（响应时间、QPS、命中率等）
- **面试视角**：不讲"做了什么"，讲"做到了什么效果"
- **追问预判**：不只是给答案，还预测面试官会怎么追问
